# 협업 규칙

## 핵심 원칙
사용자가 직접 실행하고 배울 수 있도록, AI는 **명령어나 코드를 알려주기만** 하고 **직접 실행하지 않는다**.

## 규칙

1. **명령어 실행 금지**
   - 터미널 명령어(git, pip, python 등)를 AI가 직접 실행하지 않는다.
   - 대신 실행할 명령어를 코드 블록으로 제공하고, 사용자가 직접 복사해서 실행하도록 한다.

2. **파일 수정은 OK, 실행은 NO**
   - 코드 파일(.py, .bat, .json 등) 수정은 AI가 직접 해줄 수 있다.
   - 하지만 수정 후 "이 명령어를 실행해보세요" 형식으로 안내한다.

3. **단계별 설명 필수**
   - 명령어를 알려줄 때 **왜 이 명령어를 실행하는지** 반드시 한 줄로 설명한다.
   - 예시:
     ```
     # venv 환경 안의 파이썬으로 앱을 실행합니다
     .venv\Scripts\python.exe main.py
     ```

4. **에러 발생 시**
   - 에러 메시지를 사용자가 붙여넣으면, AI가 원인을 분석하고 해결 명령어를 알려준다.
   - 사용자가 직접 해결 명령어를 실행한다.

5. **Git 작업**
   - commit, push, pr 등 git 명령어도 사용자가 직접 실행한다.
   - AI는 어떤 파일을 어떤 메시지로 커밋할지 알려준다.
